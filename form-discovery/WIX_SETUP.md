# Wix Environment Setup Instructions (F3-T1)

## Overview

This document guides you through setting up the Wix development environment for the flaviovalle.com registration system.

## Important Notes

**Modern Wix Development Workflow:**
- Wix no longer provides a local CLI for development and preview
- All Velo code development happens in the browser-based Wix Editor
- The `@wix/cli` package is only for account management
- Preview functionality is built into the Wix Editor (not CLI)

## Prerequisites

- [ ] Node.js v18+ installed
- [ ] npm installed
- [ ] A Wix account with access to flaviovalle.com
- [ ] Chrome/Firefox browser for Wix Editor

## Step 1: Verify Wix CLI Installation

The Wix CLI should already be installed globally. Verify:

```bash
wix --version
```

Expected output: Version number (e.g., `1.1.162`)

If not installed, run:

```bash
npm install -g @wix/cli
```

## Step 2: Authenticate with Wix

**MANUAL STEP REQUIRED** - Run this in your terminal (not in Claude Code):

```bash
wix login
```

This will:
1. Generate a device code (e.g., `FM3YESSP`)
2. Prompt you to open a browser
3. Ask you to paste the code at https://users.wix.com/login/device-login

**Steps:**
1. Open a **real terminal** (not Claude Code bash)
2. Run `wix login`
3. Copy the device code shown
4. Press Enter or Space as prompted
5. Browser will open to Wix login page
6. Paste the device code
7. Complete the login process

**Verify authentication:**

```bash
wix whoami
```

Expected output: Your Wix account email

## Step 3: Access flaviovalle.com Site

1. Go to https://manage.wix.com
2. Log in if needed
3. Find "flaviovalle" in your site list
4. Click on the site to open the dashboard

**Verify via CLI:**

```bash
wix site
wix list
```

Expected: Site information and list of your sites

## Step 4: Enable Dev Mode in Wix Editor

1. From the flaviovalle.com dashboard, click **"Edit Site"**
2. The Wix Editor will open in a new tab
3. Look for **"Dev Mode"** toggle in the top menu bar
4. Click "Dev Mode" → "Turn on Dev Mode"
5. A code panel should appear on the left side

**What you should see:**
- Public & Backend tabs in the code panel
- Site Structure navigator
- Properties panel
- Code files (masterPage.js, etc.)

## Step 5: Understand the Preview Workflow

**There is NO `wix preview` command.** Preview works like this:

1. Write Velo code in the Wix Editor
2. Click the **"Preview"** button in the editor (top right)
3. A new browser tab opens with your changes
4. Test your changes in the preview
5. Go back to editor to make more changes
6. Click "Publish" when ready to go live

**Preview URL:**
- Generated by Wix Editor (not CLI)
- Format: `https://editor.wix.com/html/editor/web/renderer/preview/...`
- Temporary - only valid while preview session is active
- Not shareable - tied to your editor session

## Step 6: Verify Tests Pass

Once authenticated, run the test suite:

```bash
cd form-discovery
npm test -- wix-cli-setup.test.js
```

Expected: All 12 tests should pass.

## Troubleshooting

### "Not authenticated" error

**Solution:** Run `wix login` in a real terminal (PowerShell, CMD, or Git Bash), not in Claude Code's bash.

### "Raw mode is not supported" error

**Cause:** Wix CLI requires an interactive terminal (TTY)
**Solution:** Use your system's terminal application, not embedded terminals

### "No sites found" error

**Solution:**
1. Verify you're logged into the correct Wix account
2. Check that flaviovalle.com is accessible at https://manage.wix.com
3. You may need to be invited to the site as a contributor

### "Dev Mode not found" error

**Solution:** Dev Mode is in the Wix Editor, not CLI. Follow Step 4 above.
